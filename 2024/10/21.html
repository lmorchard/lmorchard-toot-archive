Well, this was a fun bug in Knex with SQLite that I just found: Dropping a column results in all the triggers for a table also being silently dropped.

That's because they're doing that old classic thing you used to have to do with SQLite to make almost any change - i.e. create a new table with new schema, copy old table to new, delete the old table, rename the new table.

But, if you do that, you lose everything associated with the table unless you recreate it all too. They did that for indexes, but not triggers

https://github.com/knex/knex/issues/6142
